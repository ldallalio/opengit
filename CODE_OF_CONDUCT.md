# Code Of Conduct

OpenGit contributors are expected to keep project discussion professional, direct, and focused on the work.

## Expected Behavior

- Be respectful of other contributors.
- Critique code, design, and technical decisions rather than people.
- Keep issue and pull request discussion relevant to OpenGit.
- Assume mistakes are fixable and ask for clarification when intent is unclear.
- Respect privacy. Do not post private repository contents, credentials, logs with secrets, or personal information.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or behavior.
- Repeated off-topic disruption.
- Publishing another person's private information.
- Posting secrets, tokens, or private code without permission.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or access when behavior is harmful to the project or its contributors.

If you need to report conduct concerns, contact the repository maintainer privately.
